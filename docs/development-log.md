# 개발 로그

## 프로젝트 개요

Astro와 마크다운으로 슬라이드 자동 생성 시스템을 개발했습니다.

## 기술 스택

- **Astro**: 정적 사이트 생성기
- **Content Collection**: 마크다운 자동 수집
- **Paperlogy**: CDN 폰트
- **TypeScript**: 타입 안정성
- **CSS Variables**: 테마 시스템

## 개발 과정

### 1. 프로젝트 초기화
- Astro 최소 템플릿으로 프로젝트 생성
- TypeScript 설정 기본값 유지
- Git 초기화

### 2. Content Collection 설정
- `src/content/config.ts`에 슬라이드 컬렉션 정의
- `title`, `order` 필드를 포함하는 스키마 구성
- 타입 안정성 확보

### 3. 폰트 시스템
- Paperlogy 폰트 CDN 통합 (9가지 웨이트)
- CSS 변수로 폰트 시스템 구축
- font-display: swap으로 FOIT 방지

### 4. 컴포넌트 아키텍처

#### Slide.astro
- 16:9 비율의 슬라이드 컨테이너
- 중앙 정렬된 콘텐츠
- 넘침 감지를 위한 CSS 클래스

#### Sidebar.astro
- 왼쪽 고정 네비게이션
- 자동 정렬 (order 필드 기반)
- 활성 슬라이드 표시
- 홈으로 이동 링크

#### Navigation.astro
- 하단 네비게이션 바
- 이전/다음 슬라이드 링크
- 키보드 방향키 지원
- 슬라이드 카운터 표시

### 5. 스타일링 시스템

#### 마크다운 스타일링
- 헤딩 계층별 크기 및 색상
- 리스트 스타일링 (ul, ol)
- 코드 블록 하이라이팅 (Shiki dark-plus 테마)
- 테이블 스트라이프 및 호버 효과
- 인용 박스 스타일링

#### 반응형 디자인
- 모바일에서 사이드바 숨김
- 네비게이션 버튼 크기 조절
- 폰트 크기 조절

### 6. 넘침 감지 기능
- ResizeObserver API 사용
- scrollHeight와 clientHeight 비교
- 실시간 감지 및 경고 표시
- 콘솔 로그로 개발자 알림

### 7. 키보드 네비게이션
- 방향키로 슬라이드 이동
- ← ↑ : 이전 슬라이드
- → ↓ : 다음 슬라이드

## 기술적 결정

### 왜 Reveal.js를 사용하지 않았나?
- 과도한 기능 (불필요한 애니메이션, 플러그인)
- 클라이언트 사이드 초기화 오버헤드
- Astro의 정적 렌더링 장점을 활용하지 못함
- 커스터마이징 복잡도

### 왜 순수 CSS를 선택했나?
- 완전한 제어 가능
- 가벼움
- 유지보수 용이성
- Astro의 정적 빌드와 완벽한 통합

### 왜 Content Collection을 사용했나?
- 타입 안정성
- 자동 수집 및 정렬
- 프론트매터 유효성 검사
- 빌드 시간 최적화

## 구조

```
src/
├── content/
│   └── slides/           # 마크다운 슬라이드
├── components/
│   ├── Slide.astro       # 개별 슬라이드
│   ├── Sidebar.astro     # 왼쪽 네비게이션
│   └── Navigation.astro  # 하단 네비게이션
├── layouts/
│   └── Layout.astro      # 기본 레이아웃
├── pages/
│   ├── index.astro       # 홈페이지
│   └── slides/
│       └── [slug].astro # 동적 슬라이드 페이지
└── styles/
    ├── global.css        # 전역 스타일 + 폰트
    └── markdown.css      # 마크다운 스타일링
```

## 성능 특징

### 빌드 타임
- 모든 슬라이드 정적 생성
- 최소한의 클라이언트 JavaScript
- 폰트 프리로딩

### 런타임
- ResizeObserver로 넘침 감지
- 이벤트 리스너로 키보드 네비게이션
- 필요한 기능만 로드

### 번들 크기
- CSS: ~30KB
- JavaScript: ~5KB (네비게이션 + 넘침 감지)
- 폰트: CDN 로딩

## 결론

Astro와 Content Collection을 활용하여 마크다운 기반 슬라이드 시스템을 성공적으로 구축했습니다. 핵심 가치인 "마크다운만 추가하면 자동으로 슬라이드 생성"이라는 목표를 달성했으며, 개발자 경험과 최종 사용자 경험 모두 최적화되었습니다.

이 시스템은 다양한 강의 콘텐츠에 쉽게 적용할 수 있으며, 확장성과 유지보수성이 뛰어납니다.
