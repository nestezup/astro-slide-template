# 대화 기록

## 초기 요구사항

### 사용자 요청
- Astro 기반 슬라이드 서비스 개발
- 마크다운으로 강의안 작성 시 자동 슬라이드 생성
- Starlight 기본 포멧에서 왼쪽 사이드바만 유지
- 한페이지 안에 내용이 들어가는 슬라이드 (넘침 감지 및 경고)
- Paperlogy 폰트 (CDN) 적용
- docs 폴더에 기획 및 개발 정리
- 샘플 강의안: Bun으로 API 서버 만들기

### 기술적 결정

#### Reveal.js vs 순수 Astro
- **초기 계획**: Reveal.js 사용
- **최종 결정**: 순수 Astro + CSS
- **이유**: 
  - 오버엔지니어링 방지
  - Astro의 정적 렌더링 장점 유지
  - 완전한 제어권 확보
  - 더 가벼운 번들

#### Starlight vs Astro
- **명확화**: Starlight 사용하지 않고 형태만 참고
- **실제 구현**: 순수 Astro 커스텀 레이아웃

## 기술적 아이디어

### 넘침 감지 방식
- **선택**: ResizeObserver API
- **대안**: Intersection Observer, scrollHeight 비교
- **결정**: ResizeObserver가 가장 정확하고 효율적

### 폰트 로딩 전략
- **선택**: CDN + font-display: swap
- **이점**: FOIT 방지, 빠른 렌더링
- **웨이트**: 9가지 웨이트 모두 적용

### 슬라이드 네비게이션
- **방식**: URL 기반 라우팅
- **보완**: 키보드 방향키 지원
- **UI**: 하단 고정 네비게이션 바

### 마크다운 스타일링
- **코드 하이라이팅**: Shiki dark-plus 테마
- **반응형**: 모바일에서 폰트 크기 조절
- **예쁨**: 현대적인 디자인 적용

## 구조적 결정

### 파일 구조
```
src/content/slides/      # 마크다운 파일 (개수만큼 슬라이드 생성)
src/components/         # 재사용 컴포넌트
src/pages/slides/[slug].astro  # 동적 라우팅
src/styles/           # 전역 및 마크다운 스타일
```

### 자동화 시스템
- Content Collection으로 자동 수집
- getStaticPaths로 정적 경로 생성
- order 필드로 자동 정렬

### 디자인 시스템
- CSS Variables로 테마 관리
- Responsive breakpoints
- Hover states 및 transitions
